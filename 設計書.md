# 灯台カード収集管理 Web アプリ 設計書

## 1. プロジェクト概要

### 1.1 背景・目的

海上保安庁が管理する「灯台カードDigital」は、全国の灯台に掲示されたQRコードをスキャンすることで取得できるデジタルカードである。現状、取得したカードの管理は利用者個人に委ねられており、統一されたデジタル管理ツールが存在しない。本アプリはその課題を解決し、不特定多数のユーザーが自分の収集状況をオンラインで管理できる場を提供する。

### 1.2 ターゲットユーザー

- 灯台カードを収集している灯台ファン・観光客
- 全国の灯台を巡るコレクター

### 1.3 主な機能

- ユーザー登録・ログイン
- QRコードスキャンによる灯台カード収集登録
- 収集済みカードの一覧表示・管理
- 全国の灯台カード一覧・詳細閲覧
- 収集進捗の統計表示（地域別・達成率など）
- 地図上での収集状況の可視化

## 2. システム構成

### 2.1 アーキテクチャ

```
[ ブラウザ (React SPA) ]
        │  HTTP/REST API (JSON)
        ▼
[ API サーバー (FastAPI) ]
        │
        ▼
[ データベース (SQLite → PostgreSQL) ]
```

- **フロントエンド**: ブラウザ上で動作するSPAとしてReactで実装
- **バックエンド**: FastAPIによるRESTful APIサーバー
- **データベース**: 開発はSQLite、本番はPostgreSQLを想定

### 2.2 技術スタック

| 層 | 技術 | バージョン |
|---|---|---|
| フロントエンド | React + TypeScript | React 18 / TS 5 |
| フロントエンドビルド | Vite | 5.x |
| スタイリング | Tailwind CSS | 3.x |
| QRコードスキャン | html5-qrcode | 2.x |
| 地図表示 | React Leaflet | 4.x |
| バックエンド | Python + FastAPI | Python 3.11 / FastAPI 0.115 |
| ORM | SQLAlchemy | 2.x |
| データベース（開発） | SQLite | - |
| データベース（本番） | PostgreSQL | 16.x |
| 認証 | JWT (python-jose + passlib) | - |
| CORS | FastAPI middleware | - |

## 3. 機能要件

### 3.1 認証機能

| ID | 機能 | 説明 |
|---|---|---|
| AUTH-01 | ユーザー登録 | ユーザー名・メールアドレス・パスワードで新規登録 |
| AUTH-02 | ログイン | メールアドレス・パスワードでJWTトークンを取得 |
| AUTH-03 | ログアウト | クライアント側でトークンを破棄 |
| AUTH-04 | 認証保護 | コレクション管理等のAPIはJWT認証を必須とする |

### 3.2 灯台情報機能

| ID | 機能 | 説明 |
|---|---|---|
| LH-01 | 灯台一覧表示 | 全国の灯台カード一覧を地域・都道府県でフィルタリング |
| LH-02 | 灯台詳細表示 | 灯台名・場所・説明・カード画像・海上保安庁リンクを表示 |
| LH-03 | 灯台検索 | 灯台名・地域での検索 |

### 3.3 QRコードスキャン機能

| ID | 機能 | 説明 |
|---|---|---|
| QR-01 | カメラ起動 | ブラウザのカメラAPIを使いQRコードをスキャン |
| QR-02 | URL解析 | スキャンされたQRコードのURL（海上保安庁のURL）から灯台を特定 |
| QR-03 | 手動登録 | QRコードが読めない場合に灯台を手動検索して登録 |

### 3.4 コレクション管理機能

| ID | 機能 | 説明 |
|---|---|---|
| COL-01 | カード収集登録 | 灯台カードを収集済みとして登録（収集日時を自動記録） |
| COL-02 | カード一覧表示 | 収集済みカードをグリッド表示（カード画像・灯台名・収集日） |
| COL-03 | カード詳細表示 | 収集済みカードの詳細・メモの表示 |
| COL-04 | メモ追加 | 各カードに自由記述のメモを追加・編集 |
| COL-05 | 収集取消 | 誤って登録したカードを削除 |
| COL-06 | 達成率表示 | 総枚数・収集済み枚数・達成率をダッシュボードに表示 |

### 3.5 地図機能

| ID | 機能 | 説明 |
|---|---|---|
| MAP-01 | 全国地図表示 | 全灯台の位置をマーカーで地図表示 |
| MAP-02 | 収集状況の可視化 | 収集済み（緑）・未収集（グレー）でマーカーを色分け |
| MAP-03 | マーカークリック | クリックで灯台名・収集状況のポップアップを表示 |

## 4. 非機能要件

| 区分 | 要件 |
|---|---|
| セキュリティ | パスワードはbcryptでハッシュ化。JWTの有効期限は24時間。 |
| レスポンシブ | スマートフォン・PC両対応（現地でのQRスキャンを想定） |
| パフォーマンス | 灯台一覧のAPI応答は1秒以内 |
| 拡張性 | 灯台カードの追加・更新をDBへの追記のみで対応可能 |

| カラム | 型 | 制約 | 説明 |
|---|---|---|---|
| id | UUID | PK | 収集記録ID |
| user_id | UUID | FK(users.id), NOT NULL | ユーザーID |

| updated_at | TIMESTAMP | NOT NULL | 更新日時 |

## 6. API設計

### Base URL: `/api/v1`

### 6.1 認証 API

| メソッド | パス | 説明 | 認証 |
|---|---|---|---|
| POST | /auth/register | ユーザー登録 | 不要 |
| POST | /auth/login | ログイン（JWTトークン返却） | 不要 |
| GET | /auth/me | 現在のユーザー情報取得 | 必要 |

#### POST /auth/register
```json
// Request
{ "username": "taro", "email": "taro@example.com", "password": "password123" }

// Response 201
{ "id": "uuid", "username": "taro", "email": "taro@example.com", "created_at": "..." }
```

#### POST /auth/login
```json
// Request
{ "email": "taro@example.com", "password": "password123" }

// Response 200
{ "access_token": "jwt...", "token_type": "bearer" }
```

### 6.2 灯台 API

| メソッド | パス | 説明 | 認証 |
|---|---|---|---|
| GET | /lighthouses | 灯台一覧取得（フィルタ・検索対応） | 不要 |
| GET | /lighthouses/{id} | 灯台詳細取得 | 不要 |
| POST | /lighthouses/identify-by-url | QR URLから灯台を特定 | 必要 |

#### GET /lighthouses クエリパラメータ
- `region`: 地域フィルタ
- `prefecture`: 都道府県フィルタ
- `q`: 名前検索キーワード
- `collected`: `true` でログインユーザーの収集済みのみ

### 6.3 コレクション API

| メソッド | パス | 説明 | 認証 |
|---|---|---|---|
| GET | /cards | 自分の収集済みカード一覧 | 必要 |
| POST | /cards | カードを収集済みに登録 | 必要 |
| GET | /cards/{lighthouse_id} | 特定カードの収集情報取得 | 必要 |
| PATCH | /cards/{lighthouse_id} | メモを更新 | 必要 |
| DELETE | /cards/{lighthouse_id} | 収集記録を削除 | 必要 |
| GET | /cards/stats | 収集統計（達成率・地域別内訳） | 必要 |

#### POST /cards
```json
// Request
{ "lighthouse_id": 1, "note": "晴れた日に訪問" }

// Response 201
{ "id": "uuid", "lighthouse_id": 1, "collected_at": "...", "note": "..." }
```

## 9. セキュリティ設計

| 項目 | 対策 |
|---|---|
| パスワード保管 | bcrypt (コストファクター12) |
| 認証 | JWT（有効期限24時間、署名アルゴリズムHS256） |
| CORS | フロントエンドのオリジンのみ許可 |
| SQLインジェクション | SQLAlchemy ORMのパラメータバインドを使用 |
| 入力バリデーション | Pydanticによるリクエストスキーマ検証 |
